### 1. 修复安装后Web服务未自动启动的问题  
**问题描述**：安装完成后，API服务已自动启动，但Web服务（前端或后端Web应用）未启动，用户仍需手动打开系统管理工具点击“启动”。  
**预期行为**：安装程序应自动启动所有必要服务（包括Web服务），无需人工干预。  
**可能原因**：安装脚本中遗漏了Web服务的启动命令；Web服务未正确注册为Windows服务或开机自启项；启动脚本执行失败。  
**检查/修复建议**：  
- 查看安装脚本（如NSIS、Inno Setup）中是否包含启动Web服务的命令行（如 `net start DataForgeWeb` 或直接运行可执行文件）。  
- 确认Web服务已注册为Windows服务（若适用），并设置为“自动”启动类型。  
- 检查安装日志或系统事件查看器，查找启动失败的错误信息。

### 2. 修复备份计划未生成备份文件的问题  
**问题描述**：在备份管理中设置了一个计划（每天2:00执行，备份路径为 `H:1\`），但实际并未生成任何备份文件。  
**预期行为**：计划到达设定时间后，系统应在指定路径生成正确的备份文件。  
**可能原因**：备份路径格式错误（`H:1\` 应为 `H:\1` 或有效路径）；定时任务调度器未触发；备份脚本执行时发生异常；备份路径不可写或不存在。  
**检查/修复建议**：  
- 验证备份路径是否合法且可访问（如盘符存在、文件夹可写）。  
- 检查定时任务是否成功注册（如Windows任务计划程序中是否有对应任务）。  
- 查看系统或应用日志，确认备份任务是否执行及报错信息。  
- 修正路径输入验证，确保用户输入的格式正确，并给出提示。

### 3. 修复创建备份时备注不保存的问题  
**问题描述**：在备份管理页面，创建备份时填写了“备注”内容，但备份成功后，列表中的备注字段为空。  
**预期行为**：备份记录的备注应与填写的内容一致，并显示在列表中。  
**可能原因**：前端未将备注字段传递给后端；后端接口未正确保存备注；列表查询接口未返回备注字段。  
**检查/修复建议**：  
- 检查创建备份的API请求体（Payload）是否包含 `remark` 字段。  
- 检查后端控制器中是否从请求中提取了备注字段并存入数据库。  
- 检查查询备份列表的接口返回数据中是否包含备注字段，并确认前端渲染时使用了正确的字段名。

### 4. 修复报表设计列表中数据源不显示的问题  
**问题描述**：在报表设计页面的报表列表中，每一行的“数据源”列均为空白。  
**预期行为**：列表中应显示每个报表所关联的数据源名称（或其他标识）。  
**可能原因**：报表实体中未包含数据源信息；列表接口未返回数据源字段；前端渲染时使用了错误的字段名。  
**检查/修复建议**：  
- 确认报表数据库表中是否存储了数据源ID或名称，以及关联关系是否正确。  
- 检查后端查询列表的API是否返回了数据源相关信息（如 `dataSourceName`）。  
- 查看前端表格列的配置，确保绑定的字段名与返回数据一致。

### 5. 修复报表查询页面数据未实时更新的问题（钩子问题）  
**问题描述**：在报表设计页面新建报表后，切换到报表查询页面时，新报表不显示，需手动刷新页面；删除报表后，查询页面仍显示已删除的报表。  
**预期行为**：新建或删除报表后，报表查询页面应自动更新列表，无需刷新。  
**可能原因**：页面切换时未重新请求数据；组件间状态未同步（如使用Vuex/Redux但未更新）；缓存机制导致旧数据残留。  
**检查/修复建议**：  
- 检查报表查询页面组件在激活时（如 `onActivated` 或 `useEffect`）是否调用了刷新列表的方法。  
- 若使用Vue Router或React Router，确保页面切换时组件会重新挂载或触发数据获取。  
- 考虑在报表设计页执行增删操作后，通过事件总线或全局状态通知查询页面更新数据。  
- 检查是否有前端缓存（如keep-alive）导致页面不刷新，并合理配置。

### 6. 修复修改密码时错误提示“两次密码不一致”的问题  
**问题描述**：在修改密码界面，新密码和确认密码输入完全相同，但点击“确定”后仍提示“两次密码不一致”。  
**预期行为**：仅当两次输入不一致时才弹出错误提示。  
**可能原因**：前端比较逻辑错误（如比较了旧密码字段）；输入值包含不可见字符（如空格）；校验时机问题（如比较时使用了未更新的值）。  
**检查/修复建议**：  
- 检查前端表单验证代码，确认比较的是新密码和确认密码字段的值，并去除首尾空格（若业务允许）。  
- 确认输入框绑定的数据模型正确，且没有额外的转换逻辑。  
- 查看提交前的校验函数，确保在点击确定时使用了最新的输入值。
