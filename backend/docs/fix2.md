1. 修复许可证信息与实际数据不一致的问题
问题描述：许可证信息页面显示的“已使用的用户/报表/数据源”数量与数据库中的实际记录不符。
预期行为：统计数据应与数据库真实计数保持一致。
任务：检查许可证信息模块的统计逻辑，确保其从数据库获取实时数据，而非缓存或错误数据。

2. 安装完成后自动启动服务
问题描述：用户安装完成后需要手动打开系统管理工具启动服务，体验不佳。
预期行为：安装程序应在安装流程结束后自动启动 DataForgeStudio 服务。
任务：修改安装脚本（如 NSIS、Inno Setup 等），在安装完成后执行服务启动命令（如 net start DataForgeStudio）。

3. 移除不再需要的 tools\scripts 目录
问题描述：安装路径 C:\Program Files\DataForgeStudio\tools\scripts 目前可能为空或冗余，因为服务已通过安装程序注册。
预期行为：若该目录无实际用途，应从安装包中移除，减小安装包体积。
任务：确认 scripts 目录内容是否必要，如无必要则从打包脚本中剔除。

4. 优化 nssm 文件存放位置
问题描述：nssm（Non-Sucking Service Manager）相关文件目前位于 C:\Program Files\DataForgeStudio\tools\nssm，建议集中管理。
预期行为：将 nssm 文件移至 webserver 目录下（或提出更合适的存放位置），以便统一维护服务管理工具。
任务：评估并调整文件结构，更新安装脚本和服务启动脚本中的路径引用。

5. 安装后预置三个默认权限组
问题描述：根据需求，系统中应预置三个权限组（如管理员、开发者、查看者），但当前安装后未自动创建。
预期行为：数据库初始化时自动插入三个默认权限组。
任务：检查数据库迁移脚本或初始化 SQL，确保 groups 表预置三条记录，并赋予相应权限。

6. 创建默认管理员用户
问题描述：安装后缺少默认管理员账户。
预期行为：系统初始化时自动创建用户 admin，密码 Admin@123，且具有最高权限。
任务：在数据库种子脚本中添加该用户记录（注意密码需加密存储）。

7. 修复数据源编辑弹窗信息显示不全的问题
问题描述：在“数据源管理”页面，点击已存在数据源的“编辑”按钮，弹出的弹窗未显示所有字段。
预期行为：弹窗应完整展示以下字段：数据源名称、数据库类型、服务器地址、端口、用户名、密码、数据库名、描述。
任务：修改前端代码，使编辑弹窗正确加载并回显所有字段值；同时检查后端接口是否返回完整数据。

8. 修复报表编辑弹窗信息显示不全的问题
问题描述：在“报表设计”页面，点击已保存报表的“编辑”按钮，弹出的界面缺失部分字段。
预期行为：弹窗应完整展示：报表名称、分类、描述、数据源、SQL语句、字段配置、查询条件、视图配置。
任务：修正前端编辑弹窗的渲染逻辑，确保所有字段正确回显；后端需保证返回完整的报表定义数据。

9. 修复新增报表时表单残留数据的问题
问题描述：点击“新增报表”按钮后，表单中仍显示之前编辑过的报表数据。
预期行为：新增操作应打开空白表单，所有字段均为默认值。
任务：检查新增按钮的事件处理，确保每次打开新增弹窗前重置表单数据。

10. 修复首页仪表盘数据统计不准确的问题
问题描述：首页显示的仪表数据（如用户数、报表数等）与实际数据库记录不一致。
预期行为：仪表盘数据应与数据库实时计数同步。
任务：审查首页数据获取接口，排除缓存或错误聚合逻辑，确保每次刷新都查询最新数据。

11. 改进备份创建交互：使用文件夹选择对话框
问题描述：当前“创建备份”按钮左侧为路径输入框，用户需手动输入路径，易出错且不直观。
预期行为：点击“创建备份”按钮后，弹出系统文件夹树选择窗口，用户可浏览并选择任意分区下的文件夹作为备份存放路径，确认后立即执行备份。
任务：将输入框替换为触发文件夹选择对话框的按钮，或集成一个可点击的图标，调起系统文件对话框，并将选中路径传入备份逻辑。

12. 改进备份计划路径选择交互
问题描述：在“备份管理-备份计划”的“新增计划”界面中，备份路径输入框右侧仅有说明文字，无法便捷选择路径。
预期行为：将说明文字改为“选择”按钮，点击后弹出文件夹树选择窗口，用户选中文件夹后自动将路径填入输入框。
任务：修改界面，添加按钮并绑定文件夹选择事件，实现路径自动填充。
