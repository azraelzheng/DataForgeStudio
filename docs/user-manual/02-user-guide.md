# 用户管理指南

## 用户列表

### 访问路径
**系统管理** → **用户管理**

### 功能说明
- 查看所有用户列表（root 用户除外）
- 创建、编辑、删除用户
- 重置用户密码
- 分配用户角色

### 用户列表字段

| 字段 | 说明 |
|------|------|
| 用户名 | 登录系统使用的用户名 |
| 真实姓名 | 用户的真实姓名 |
| 邮箱 | 联系邮箱 |
| 部门 | 所属部门 |
| 职位 | 职位名称 |
| 状态 | 激活/锁定/禁用 |
| 角色 | 用户拥有的角色 |

---

## 新增用户

### 步骤

1. 点击 **新增用户** 按钮
2. 填写用户信息：

#### 基本信息
- **用户名** (必填): 登录系统使用的用户名，必须唯一
- **真实姓名**: 用户的真实姓名
- **邮箱**: 联系邮箱地址
- **手机号**: 联系电话
- **部门**: 所属部门
- **职位**: 职位名称

#### 账户设置
- **密码**: 初始密码（可选，留空则生成随机密码）
- **强制修改密码**: 下次登录时必须修改密码
- **账户状态**: 激活/禁用

#### 权限设置
- **角色**: 为用户分配一个或多个角色

3. 点击 **保存** 完成创建

---

## 编辑用户

### 步骤

1. 在用户列表中找到要编辑的用户
2. 点击 **编辑** 按钮
3. 修改用户信息
4. 点击 **保存** 提交更改

### 可修改内容
- 真实姓名、邮箱、手机号
- 部门、职位
- 账户状态
- 用户角色

> **注意**: 用户名创建后不可修改。

---

## 重置密码

### 步骤

1. 在用户列表中找到目标用户
2. 点击 **重置密码** 按钮
3. 系统会自动生成一个随机的安全密码
4. 将新密码告知用户

### 密码要求
- 最小长度: 6 位
- 建议包含: 大小写字母、数字、特殊字符
- 系统使用 BCrypt 加密存储密码

---

## 删除用户

### 步骤

1. 在用户列表中找到要删除的用户
2. 点击 **删除** 按钮
3. 确认删除操作

### 限制条件
- **root 用户**不可删除（IsSystem=1）
- 用户名删除后不可恢复
- 如果用户有关联数据（如创建的报表），删除前请确认

---

## 分配角色

### 角色说明

角色是权限的集合，通过为用户分配角色来控制其访问权限。

### 预定义角色

| 角色名 | 说明 |
|--------|------|
| 超级管理员 | 拥有所有权限 |
| 管理员 | 系统管理权限 |
| 普通用户 | 基础查看权限 |
| 报表设计师 | 报表设计和导出权限 |
| 只读用户 | 仅查看权限，无导出 |

### 分配步骤

1. 编辑用户
2. 在 **角色** 下拉框中选择一个或多个角色
3. 保存更改

### 权限叠加
用户可以拥有多个角色，其权限为所有角色权限的并集。

---

## 用户状态

### 状态类型

| 状态 | 说明 | 行为 |
|------|------|------|
| 激活 | 正常状态 | 可以正常登录 |
| 禁用 | 管理员禁用 | 无法登录，提示"用户已被禁用" |
| 锁定 | 密码错误次数过多 | 无法登录，提示"账户已被锁定" |

### 解锁用户

如果用户被锁定，管理员可以：

1. 编辑用户
2. 将状态改为 **激活**
3. 保存更改

这将重置密码错误计数并解锁账户。

---

## root 用户说明

### 特殊属性
- **IsSystem = 1**: 标记为系统用户
- **不可见**: 不在用户列表中显示
- **不可删除**: 任何删除操作都会被阻止
- **所有权限**: 自动拥有所有权限，无需显式分配

### 使用建议
- 仅用于系统维护和紧急情况
- 日常管理使用普通管理员账户
- 定期更换 root 密码

---

## 最佳实践

### 密码管理
1. 定期要求用户修改密码
2. 使用"强制修改密码"功能确保新用户首次登录时修改密码
3. 重置密码后生成强密码并通过安全渠道告知用户

### 权限分配
1. 遵循最小权限原则
2. 使用角色而非直接分配权限
3. 定期审查用户权限，移除不必要的权限

### 账户维护
1. 定期清理不活跃用户
2. 禁用而非删除离职用户的账户（保留历史数据）
3. 监控异常登录行为

## 常见问题

### Q: 用户登录后看不到某些菜单？

**A**: 检查用户的角色是否具有相应权限。在 **角色管理** 中查看角色的权限配置。

### Q: 如何批量导入用户？

**A**: 当前版本不支持批量导入。如需此功能，请联系技术支持。

### Q: 用户密码错误几次后会被锁定？

**A**: 连续 5 次密码错误后账户会被自动锁定。管理员需要手动解锁。
