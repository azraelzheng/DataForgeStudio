## 前置条件

### 1. 确保密钥文件存在

许可证系统依赖 RSA 密钥对进行签名。首次运行后端服务时，系统会自动在以下位置生成密钥：

```
backend/src/DataForgeStudio.Api/keys/
├── private_key.pem    # 私钥（用于签名，必须保密）
└── public_key.pem     # 公钥（用于验证，部署到客户服务器）
```

**重要**：
- 私钥文件必须妥善保管，不要泄露
- 如果密钥丢失，需要重新生成并通知所有客户更新许可证

### 2. 安装 .NET SDK

确保已安装 .NET 8.0 SDK：

```bash
dotnet --version  # 应显示 8.0.x
```

## 获取客户机器码

### 方法一：通过系统管理界面

1. 客户登录 DataForgeStudio
2. 进入 **系统管理** → **系统设置**
3. 在页面中找到 **机器码** 信息
4. 客户将机器码发送给你

### 方法二：通过 API 获取

客户可以在浏览器中访问：

```
http://<服务器地址>:<端口>/api/system/machine-code
```

返回示例：
```json
{
  "success": true,
  "data": "A1B2C3D4E5F6G7H8I9J0K1L2M3N4O5P6"
}
```

## 生成许可证

### 步骤 1：打开许可证生成工具

在项目根目录下执行：

```bash
cd backend/tools/LicenseGenerator
dotnet run
```

### 步骤 2：输入许可证信息

按照提示依次输入：

#### 2.1 客户名称
```
请输入客户名称: 张三公司
```
- 建议使用客户公司名称
- 名称会显示在系统许可证信息中

#### 2.2 许可证类型
```
许可证类型:
  1. Trial         (试用版)
  2. Standard      (标准版) - 默认
  3. Professional  (专业版)
  4. Enterprise    (企业版)
请选择许可证类型 (1-4, 默认: 2):
```

各类型默认限制：

| 类型 | 有效期 | 最大用户 | 最大报表 | 最大数据源 |
|------|--------|----------|----------|------------|
| Trial | 30天 | 5 | 10 | 2 |
| Standard | 1年 | 20 | 50 | 5 |
| Professional | 1年 | 100 | 无限制 | 无限制 |
| Enterprise | 2年 | 无限制 | 无限制 | 无限制 |

#### 2.3 过期日期
```
过期日期 (默认: 2026-02-18)
请输入过期日期 (格式: yyyy-MM-dd, 直接回车使用默认值): 2027-02-18
```

#### 2.4 最大用户数
```
最大用户数 (默认: 20): 50
```
- 输入数字，根据合同约定
- 输入 0 表示无限制（仅 Professional 和 Enterprise）

#### 2.5 最大报表数
```
最大报表数 (默认: 50, 输入 0 表示无限制): 100
```

#### 2.6 最大数据源数
```
最大数据源数 (默认: 5, 输入 0 表示无限制): 10
```

#### 2.7 功能选择
```
可用功能:
  1. 报表设计
  2. 报表查询
  3. 图表展示
  4. Excel导出
  5. PDF导出
  6. 数据源管理
  7. 用户管理
  8. 角色管理

请输入启用的功能编号 (多个用逗号分隔, 如: 1,2,3,5): 1,2,3,4,5,6
```
- 直接回车：启用所有功能
- 输入编号：选择特定功能

#### 2.8 机器码绑定
```
机器码绑定 (可选)
如需绑定到特定服务器，请输入机器码；留空则不绑定
机器码 (留空则不绑定): A1B2C3D4E5F6G7H8I9J0K1L2M3N4O5P6
```
- **重要**：输入客户提供的机器码
- 如果不绑定机器码，许可证可在任意服务器使用（不推荐）

### 步骤 3：确认信息

系统会显示预览：
```
====================================================
许可证信息预览:
  客户名称: 张三公司
  许可证类型: Standard
  过期日期: 2027-02-18
  最大用户数: 50
  最大报表数: 100
  最大数据源数: 10
  启用功能: 报表设计, 报表查询, 图表展示, Excel导出, PDF导出, 数据源管理
  绑定机器码: A1B2C3D4E5F6G7H8I9J0K1L2M3N4O5P6
====================================================

确认生成许可证? (Y/N): Y
```

输入 `Y` 确认生成。

### 步骤 4：获取许可证文件

生成成功后，许可证文件位于：

```
backend/tools/LicenseGenerator/licenses/
└── 张三公司_Standard_20260218170800.lic
```

文件名格式：`{客户名称}_{许可证类型}_{时间戳}.lic`

## 交付许可证

### 方式一：发送文件

1. 将 `.lic` 文件发送给客户（邮件、网盘等）
2. 客户在系统中导入许可证

### 方式二：客户自行导入

客户操作步骤：

1. 登录 DataForgeStudio
2. 进入 **系统管理** → **许可证管理**
3. 点击 **导入许可证**
4. 选择 `.lic` 文件
5. 系统自动验证并激活

### 通过 API 激活（高级）

如果客户需要通过 API 激活：

```bash
curl -X POST http://<服务器地址>:<端口>/api/license/activate \
  -H "Content-Type: application/json" \
  -d '{"licenseKey": "许可证文件内容"}'
```

许可证文件内容 = 打开 `.lic` 文件，复制全部文本

## 常见问题

### Q1: 许可证激活失败，提示"机器码不匹配"

**原因**：许可证绑定的机器码与当前服务器不匹配

**解决**：
1. 让客户重新获取当前服务器的机器码
2. 使用正确的机器码重新生成许可证

### Q2: 许可证激活失败，提示"已被篡改"

**原因**：许可证文件被修改或损坏

**解决**：
1. 确保许可证文件未经过任何编辑
2. 重新生成许可证
3. 使用二进制方式传输文件（避免编码问题）

### Q3: 许可证激活失败，提示"已过期"

**原因**：许可证过期日期早于当前日期

**解决**：
1. 检查服务器系统时间是否正确
2. 重新生成有效期更长的许可证

### Q4: 如何查看当前许可证信息？

在系统管理界面：
1. 进入 **系统管理** → **许可证管理**
2. 查看许可证详情

通过 API：
```bash
GET /api/license
```

### Q5: 如何更新许可证？

1. 生成新的许可证文件
2. 客户导入新许可证
3. 新许可证会自动替换旧许可证

### Q6: 试用期是如何工作的？

- 首次运行系统时，自动生成 15 天试用期
- 试用期结束后，必须导入正式许可证
- 试用期绑定机器，不能重复使用

### Q7: 许可证可以转移吗？

- 未绑定机器码的许可证：可以转移（不推荐）
- 已绑定机器码的许可证：只能在绑定的服务器使用
- 如需转移，请重新生成许可证

## 许可证类型详细说明

### Trial（试用版）

- **适用场景**：产品试用、功能演示
- **有效期**：30 天
- **限制**：
  - 最多 5 个用户
  - 最多 10 个报表
  - 最多 2 个数据源
  - 基础功能（报表设计、查询、图表、导出）

### Standard（标准版）

- **适用场景**：中小型企业
- **有效期**：1 年
- **限制**：
  - 最多 20 个用户
  - 最多 50 个报表
  - 最多 5 个数据源
  - 标准功能（不含高级管理功能）

### Professional（专业版）

- **适用场景**：大型企业、开发团队
- **有效期**：1 年
- **限制**：
  - 最多 100 个用户
  - 无限报表
  - 无限数据源
  - 所有功能

### Enterprise（企业版）

- **适用场景**：企业级部署
- **有效期**：2 年
- **限制**：
  - 无限用户
  - 无限报表
  - 无限数据源
  - 所有功能
  - 优先技术支持

## 安全建议

1. **私钥保护**
   - 不要将私钥文件提交到版本控制
   - 定期备份私钥到安全位置
   - 限制私钥文件的访问权限

2. **许可证管理**
   - 记录每个许可证的生成信息
   - 建立许可证台账
   - 定期检查许可证使用情况

3. **机器码验证**
   - 建议所有正式许可证都绑定机器码
   - 避免许可证被复制到其他服务器
