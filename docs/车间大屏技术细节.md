基于你的技术栈（[ASP.NET](http://asp.net/) Core 8.0 后端、Vue 3 前端、SQL Server 2005+），下面详细描述两个阶段的技术核心内容及实现方法。

---

## 阶段一：基于已发布的自定义报表

### 核心目标

复用现有自定义报表模块，通过看板设计器选择报表、绑定字段，实现可视化展示。

### 技术架构概览

- **后端**：[ASP.NET](http://asp.net/) Core 8.0 提供 REST API，包括报表元数据获取、报表数据查询、看板定义存储。
- **前端**：Vue 3 实现拖拽画布（使用 `vue-draggable-plus` 或 `grid-layout`）、组件配置面板、图表渲染（ECharts / AntV）。
- **数据库**：SQL Server 存储看板定义（JSON 格式）、报表元数据缓存（可选）。

### 详细实现

### 1. 报表元数据接口

**目的**：当用户选择某个报表时，前端需要获取该报表的输出字段列表（名称、类型），用于字段绑定。

**后端实现**：

- 假设报表模块已有 `Report` 实体，包含报表定义（可能存储为 SQL 文本、存储过程名或报表引擎的内部表示）。
- 新增 API：`GET /api/reports/{id}/metadata`。
- 内部逻辑：根据报表类型，解析输出字段。
    - 如果报表是存储过程，可使用 `SET FMTONLY ON` 或 `sp_describe_first_result_set` 获取元数据（注意 SQL Server 2005 可能不支持后者，可使用 `OpenQuery` 或执行 `SELECT TOP 0 * FROM ...` 来获取列信息）。
    - 如果报表是 SQL 查询，可对查询包装 `SELECT TOP 0 * FROM ({query}) AS t` 执行，从结果集的架构中获取列名和数据类型。
- 返回 JSON 数组，每个元素包含 `fieldName`、`dataType`（string、number、date、boolean）。

**代码示例**（C#）：

```csharp
[HttpGet("{id}/metadata")]
public async Task<ActionResult<IEnumerable<FieldMetadata>>> GetReportMetadata(int id)
{
    var report = await _reportRepository.GetByIdAsync(id);
    if (report == null) return NotFound();

    var sql = $"SELECT TOP 0 * FROM ({report.Query}) AS t";
    using var connection = new SqlConnection(_connectionString);
    var command = new SqlCommand(sql, connection);
    await connection.OpenAsync();
    using var reader = await command.ExecuteReaderAsync(CommandBehavior.SchemaOnly);
    var schemaTable = reader.GetSchemaTable();
    var fields = schemaTable.Rows.Cast<DataRow>().Select(row => new FieldMetadata
    {
        FieldName = row["ColumnName"].ToString(),
        DataType = MapSqlTypeToDataType(row["DataType"].ToString())
    });
    return Ok(fields);
}
```

### 2. 报表数据接口

**目的**：根据报表 ID 和参数，返回标准格式的数据（`{columns, rows}`）。

**后端实现**：

- API：`POST /api/reports/{id}/data`（支持参数传递）。
- 请求体包含参数键值对（从看板过滤器映射而来）。
- 执行报表查询（注意参数化防止 SQL 注入），将结果转换为标准格式：
    - `columns`：字段名数组。
    - `rows`：二维数组，每行对应一条记录。
- 考虑缓存：使用 `IMemoryCache` 或 `IDistributedCache` 对相同报表+参数组合的结果缓存一段时间（例如 5 分钟）。

**代码示例**：

```csharp
[HttpPost("{id}/data")]
public async Task<ActionResult<ReportData>> GetReportData(int id, [FromBody] Dictionary<string, object> parameters)
{
    var report = await _reportRepository.GetByIdAsync(id);
    // 替换查询中的参数占位符（如 @startDate），或使用存储过程参数
    var sql = ReplaceParameters(report.Query, parameters);
    using var connection = new SqlConnection(_connectionString);
    var dataTable = new DataTable();
    using var adapter = new SqlDataAdapter(sql, connection);
    adapter.Fill(dataTable);

    var columns = dataTable.Columns.Cast<DataColumn>().Select(c => c.ColumnName).ToArray();
    var rows = dataTable.Rows.Cast<DataRow>().Select(r => columns.Select(c => r[c]).ToArray()).ToArray();
    return Ok(new ReportData { Columns = columns, Rows = rows });
}
```

### 3. 看板定义存储

**目的**：保存用户设计的看板（包含组件布局、数据源配置）。

**数据结构**：在 SQL Server 中创建 `Dashboards` 表，包含 `Id`、`Name`、`Definition`（JSON 字段）、`CreatedBy`、`CreatedAt` 等。

`Definition` JSON 示例：

```json
{
  "layout": [
    { "i": "chart1", "x": 0, "y": 0, "w": 6, "h": 4 }
  ],
  "components": {
    "chart1": {
      "type": "bar",
      "dataSource": {
        "type": "report",
        "reportId": 123,
        "mappings": [
          { "target": "xAxis", "sourceField": "order_date" },
          { "target": "yAxis", "sourceField": "amount", "aggregate": "sum" }
        ],
        "filters": [ /* 过滤条件 */ ],
        "parameterMappings": {
          "globalDateRange": "startDate"  // 将看板全局日期范围映射到报表参数 startDate
        }
      }
    }
  }
}
```

### 4. 前端实现要点

- **拖拽画布**：使用 `vue-grid-layout` 或 `vue-draggable-next` 实现组件的拖拽布局。
- **组件库**：预定义图表组件（封装 ECharts），接收 `data` 和 `mappings` 作为 props，内部调用适配器转换数据后渲染。
- **配置面板**：当选中组件时，右侧面板显示配置项。
    - 数据源选择：下拉框列出用户有权限的自定义报表。
    - 字段绑定：根据所选报表的元数据，动态生成字段选择器（如 X 轴选择器、Y 轴选择器）。
    - 参数映射：如果报表有参数，提供看板全局过滤器与报表参数的映射配置。
- **数据适配器**：编写一个适配器函数，将后端返回的标准 `{columns, rows}` 转换为 ECharts 所需的格式。在组件 mounted 或 data 变化时调用。

**适配器示例（Vue 3）**：

```jsx
function adaptToECharts(data, mappings, chartType) {
  // data: { columns, rows }
  // mappings: 如 { xAxis: 'order_date', yAxis: 'amount' }
  const xField = mappings.xAxis;
  const yField = mappings.yAxis;
  const xData = rows.map(row => row[columns.indexOf(xField)]);
  const yData = rows.map(row => row[columns.indexOf(yField)]);
  return {
    xAxis: { data: xData },
    series: [{ type: chartType, data: yData }]
  };
}
```

### 5. 权限考虑

- 报表权限复用现有报表模块的权限体系，看板仅能选择用户有权访问的报表。
- 看板本身的查看权限：可在 Dashboard 表中增加权限字段或通过角色控制。

## 阶段二：直接连接原始数据表

### 核心目标

提供灵活的数据探索能力，允许用户直接连接 SQL Server 表、视图或编写自定义 SQL，自由组合字段创建图表。

### 新增技术核心

### 1. 元数据获取接口

**目的**：列出数据库中的表/视图及其字段。

**后端实现**：

- API：`GET /api/metadata/tables` 返回所有表名（可限定架构）。
- API：`GET /api/metadata/tables/{tableName}/columns` 返回表的字段列表（名称、类型、是否可为空等）。
- 使用 SQL Server 系统视图：`INFORMATION_SCHEMA.TABLES` 和 `INFORMATION_SCHEMA.COLUMNS`。

**注意**：SQL Server 2005 支持 `INFORMATION_SCHEMA`，可以兼容。

### 2. 数据集管理

**目的**：允许用户将多个表关联，定义计算字段，形成逻辑数据集，供看板使用。

**后端设计**：

- 创建 `DataSet` 实体，包含 `Id`、`Name`、`Tables`（JSON 存储参与的表及关联关系）、`Fields`（JSON 存储字段定义，包括物理字段、计算字段表达式）、`Parameters`（可选参数）等。
- 提供 CRUD API 管理数据集。
- 查询时，根据数据集定义动态生成 SQL。

**数据集 JSON 示例**：

```json
{
  "tables": [
    { "alias": "o", "name": "Orders", "type": "table" },
    { "alias": "od", "name": "OrderDetails", "type": "table", "join": { "type": "left", "on": "o.OrderID = od.OrderID" } }
  ],
  "fields": [
    { "name": "OrderDate", "expression": "o.OrderDate", "type": "date" },
    { "name": "ProductID", "expression": "od.ProductID", "type": "int" },
    { "name": "Revenue", "expression": "od.Quantity * od.UnitPrice", "type": "decimal", "isCalculated": true }
  ]
}
```

### 3. 查询生成器

**目的**：根据用户在前端拖拽的字段（维度、度量）、过滤条件，生成 SQL 查询语句。

**后端实现**：

- 接收前端传来的查询配置（数据集 ID、选择的字段、聚合方式、过滤条件、排序等）。
- 解析数据集定义，构建 SELECT 语句：
    - 维度字段放入 GROUP BY。
    - 度量字段使用聚合函数（SUM、AVG、COUNT 等）。
    - 过滤条件转换为 WHERE 子句（参数化）。
- 注意 SQL Server 2005 的语法限制（如不支持 OFFSET FETCH，需用 ROW_NUMBER 实现分页）。

**安全**：使用参数化查询防止 SQL 注入；对字段名、表名使用 `SqlConnection.DelimitIdentifier` 进行转义。

### 4. 查询执行引擎

**目的**：执行生成的 SQL，返回数据。

**后端实现**：

- 与阶段一类似，使用 [ADO.NET](http://ado.net/) 执行 SQL，返回 `{columns, rows}`。
- 添加超时控制和查询取消支持。
- 引入缓存：对相同的查询（哈希化）结果缓存一段时间，适用于变化不频繁的数据。

### 5. 适配器增强

阶段一的适配器仍可使用，但需支持更复杂的映射，例如：

- 多系列图表（如分组柱状图）需要将维度字段映射到系列，此时返回的数据结构可能是扁平表，适配器需将其转换为 ECharts 所需的 series 数组。
- 可能需要在适配器中实现简单的数据透视。

**增强适配器示例**（处理分组柱状图）：

```jsx
function adaptGroupedBar(data, mappings) {
  // mappings: { xAxis: 'category', series: 'product', yAxis: 'sales' }
  const xField = mappings.xAxis;
  const seriesField = mappings.series;
  const yField = mappings.yAxis;

  // 获取所有系列值
  const seriesSet = [...new Set(data.rows.map(r => r[data.columns.indexOf(seriesField)]))];
  // 获取所有 X 轴值
  const xSet = [...new Set(data.rows.map(r => r[data.columns.indexOf(xField)]))];

  const series = seriesSet.map(seriesName => {
    const seriesData = xSet.map(xVal => {
      const row = data.rows.find(r =>
        r[data.columns.indexOf(xField)] === xVal &&
        r[data.columns.indexOf(seriesField)] === seriesName
      );
      return row ? row[data.columns.indexOf(yField)] : 0;
    });
    return { name: seriesName, type: 'bar', data: seriesData };
  });

  return { xAxis: { data: xSet }, series };
}
```

### 6. 权限控制

- **行级权限**：在查询生成时，根据当前用户动态追加过滤条件（如 `WHERE department = @userDept`）。可通过在数据集定义中关联权限规则实现。
- **列级权限**：在字段列表中过滤掉用户无权查看的字段。

### 7. 前端增强

- **数据集选择器**：让用户选择已创建的数据集。
- **字段拖拽绑定**：提供类似 Tableau 的“行/列”架子，用户可将字段拖入“维度”、“度量”区域，自动生成图表。
- **高级过滤**：支持构建复杂的过滤条件（AND/OR 组合）。
- **图表联动**：通过事件总线或全局状态管理，实现点击图表触发其他图表刷新（传递过滤条件）。

## 两个阶段的平滑过渡策略

1. **统一数据源抽象**：在阶段一设计时，定义一个 `IDataSource` 接口，包含 `GetMetadataAsync` 和 `GetDataAsync` 方法。阶段一的报表数据源实现该接口；阶段二的数据库表/数据集也实现该接口。这样前端配置面板可以统一处理不同类型的数据源，只需在数据源类型选择时切换不同的配置 UI。
2. **共享画布与组件**：阶段一的拖拽画布、图表组件、布局存储完全复用，阶段二仅增强数据绑定能力。
3. **配置面板渐进增强**：
    - 阶段一：简单字段选择（X轴、Y轴）。
    - 阶段二：增加“高级模式”复选框，展开后显示维度/度量区域、聚合函数选择、计算字段等。
4. **权限模型统一**：将报表权限和数据集权限都接入统一的权限服务，看板权限基于角色控制。
5. **数据格式兼容**：阶段二的查询引擎输出与阶段一报表数据接口相同的标准 `{columns, rows}` 格式，确保所有图表组件和适配器无需改动即可工作。

## 技术栈细节考虑

- [**ASP.NET](http://asp.net/) Core 8.0**：使用最新的 LTS 版本，支持原生 AOT（可选），性能优秀。使用 `System.Text.Json` 处理 JSON。
- **Vue 3**：使用 Composition API + TypeScript 提高可维护性。状态管理可使用 Pinia。UI 库可选 Element Plus 或 Ant Design Vue。
- **SQL Server 2005+**：注意某些新特性（如 OFFSET FETCH、窗口函数）在 2005 中不可用。分页需使用 `ROW_NUMBER()` 或 `TOP` 配合嵌套查询。参数化查询是必须的。
- **拖拽库**：推荐 `vue3-draggable-resizable` 或 `grid-layout` 的 Vue 3 版本。
- **图表库**：ECharts 5 或 AntV G2Plot，都有 Vue 3 封装。
