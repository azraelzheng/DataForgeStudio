# 报表查询页面布局优化设计

**日期**: 2026-02-18
**状态**: 已批准

---

## 需求

优化报表查询页面的 UI 布局：
- 减少报表列表和查询条件的留白
- 让查询结果占用更多空间
- 实现查询条件的智能折叠功能

---

## 设计方案

### 1. 报表列表侧边栏优化

| 项目 | 当前值 | 优化后 |
|------|--------|--------|
| 侧边栏宽度 | 280px | 220px |
| 报表项内边距 | 12px | 8px |
| 列表区域边距 | 8px | 4px |
| 报表项间距 | 4px + 4px margin | 2px gap |
| 搜索框区域高度 | 56px | 48px |
| 搜索框内边距 | 12px | 8px |

### 2. 查询条件区域优化

| 项目 | 当前值 | 优化后 |
|------|--------|--------|
| 卡片样式 | 有边框和阴影 | 扁平化，无边框 |
| 内边距 | 16px | 10px |
| 标题行高度 | 44px | 36px |

### 3. 智能折叠功能（新增）

**交互逻辑：**
- **初始状态**：所有查询条件展开显示
- **点击"查询"按钮后**：只显示第一行条件，隐藏其余条件，结果区域自动扩展
- **点击折叠按钮**：可以在"全部条件"和"首行条件"之间切换

**实现要点：**
- 计算第一行能容纳的条件数量（基于屏幕宽度）
- 折叠状态时隐藏超出第一行的条件
- 使用 flex 布局让结果区域自动填充剩余空间

### 4. 结果区域自适应

- 使用 `flex: 1` 和 `min-height: 0` 实现自动填充
- 条件折叠时，结果区域自动变高
- 条件展开时，结果区域自动变矮

## 预期效果

- 侧边栏减少 60px 宽度，主内容区域增加相应空间
- 查询条件区域高度减少约 20-30px
- 智能折叠功能可在查询后节省约 50-100px 高度（取决于条件数量）
- 整体查询结果区域可增加约 10-15% 的显示空间
