1. 进入备份管理提示“服务器内部错误”。控制台日志显示“GET http://127.0.0.1:8089/api/system/backup-schedules 500 (Internal Server Error)”
2. 不输入备份路径直接点击创建备份，提示“数据库备份失败: SQL错误 3201: 无法打开备份设备 'C:\\Program Files\\DataForgeStudio\\Server\\Backups\\DataForgeStudio_20260224_194647.bak'。出现操作系统错误 5(拒绝访问。)。 BACKUP DATABASE 正在异常终止。 已将数据库上下文更改为 "master"。”
3. 输入备份路径，点击创建备份后备份列表显示两个备份文件
4. 创建备份计划，点击保存时提示“服务器内部错误”，控制台提示“POST http://127.0.0.1:8089/api/system/backup-schedules 500 (Internal Server Error)”
5. 修改创建备份和备份计划中备份路径的输入方式，不是直接在输入框中输入备份路径。
    
    5.1 点击创建备份弹出文件夹选择框，选中具体文件夹后点确定开始备份。
    
    5.2 备份计划中备份路径右边增加按钮，点击后与创建备份一样弹出文件夹选择框，选中具体的文件夹后点确定，将选中的文件夹路径回填路径输入框中。
    
6. 初始化数据库时，缺少默认的权限组和admin用户（密码Admin@123）
7. 日志文件夹（C:\Program Files\DataForgeStudio\logs）只有configurator.log，没有API服务运行的错误日志，没有web服务运行错误日志。
8. 数据源管理中数据源列表的创建时间格式应该与备份列表的时间格式显示一致。
9. 权限组管理中的权限组列表的创建时间格式应该与备份列表的时间格式显示一致。
10. 报表设计中报表列表的创建时间格式应该与备份列表的时间格式显示一致。
