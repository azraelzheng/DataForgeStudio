## 1. 产品简介

### 1.1 产品概述

DataForgeStudio 是一款面向企业的智能报表设计与数据集成平台，旨在帮助企业快速构建、管理和发布动态报表，实现多数据源的统一查询与分析。

### 1.2 核心功能

| 功能模块 | 描述 |
|---------|------|
| 报表设计器 | 可视化SQL设计器，支持参数化查询 |
| 报表查询 | 在线查询与数据导出 |
| 数据源管理 | 多数据库连接管理 |
| 用户管理 | 用户账户与角色权限 |
| 许可证管理 | 软件授权与试用期管理 |
| 系统备份 | 数据备份与恢复 |
| 操作日志 | 系统操作记录 |

### 1.3 支持的数据库

- SQL Server 2005+
- MySQL 5.7+
- PostgreSQL 10+
- Oracle 11g+
- SQLite 3.x

## 2. 系统安装

### 2.1 系统要求

| 项目 | 最低要求 | 推荐配置 |
|------|---------|---------|
| 操作系统 | Windows Server 2012 / Windows 10 | Windows Server 2019 / Windows 11 |
| CPU | 2核 | 4核+ |
| 内存 | 4GB | 8GB+ |
| 磁盘空间 | 500MB | 2GB+ |
| 数据库 | SQL Server 2005+ | SQL Server 2016+ |
| .NET | .NET 8.0 Runtime | .NET 8.0 Runtime |

### 2.2 安装步骤

1. **运行安装程序**
   - 双击 `DataForgeStudio-Setup.exe`
   - 选择安装目录（默认：`C:\Program Files\DataForgeStudio`）

2. **配置数据库连接**
   - 输入SQL Server服务器地址
   - 输入数据库端口（默认：1433）
   - 选择Windows身份验证或SQL Server身份验证

3. **完成安装**
   - 安装程序会自动创建Windows服务
   - 服务名称：`DFAppService`（API服务）、`DFWebService`（Web服务）

4. **访问系统**
   - 打开浏览器访问：`http://localhost:8089`
   - 使用 root 账户登录（首次密码查看安装日志）

### 2.3 安装后验证

1. 检查Windows服务是否正常运行
2. 访问Web界面确认登录页面正常显示
3. 使用root账户登录验证功能正常

## 3. 快速开始

### 3.1 首次登录

1. 打开浏览器，访问 `http://localhost:8089`
2. 输入用户名：`root`
3. 输入密码（安装时生成的临时密码）
4. 点击"登录"按钮

> **重要：** 首次登录后请立即修改密码！

### 3.2 修改密码

1. 登录后点击右上角用户头像
2. 选择"修改密码"
3. 输入当前密码和新密码
4. 点击"确认"保存

### 3.3 创建第一个报表

1. 进入"报表管理" → "报表设计"
2. 点击"新建报表"按钮
3. 填写报表基本信息：
   - 报表名称
   - 报表描述
   - 选择数据源
4. 编写SQL查询语句
5. 添加查询参数（可选）
6. 点击"保存"按钮
7. 进入"报表查询"测试报表

## 4. 功能模块

### 4.1 报表设计器

#### 4.1.1 创建报表

1. 导航至"报表管理" → "报表设计"
2. 点击"新建报表"
3. 填写报表信息：
   - **报表名称**：便于识别的名称
   - **报表描述**：功能说明
   - **数据源**：选择已配置的数据源
   - **SQL语句**：查询语句（仅支持SELECT）

#### 4.1.2 SQL参数化

支持在SQL中使用参数，格式：`@参数名`

示例：
```sql
SELECT * FROM Orders
WHERE OrderDate BETWEEN @StartDate AND @EndDate
AND CustomerId = @CustomerId
```

参数配置：
- **参数名**：与SQL中@后面的名称一致
- **参数类型**：文本、数字、日期、下拉选择
- **默认值**：可选
- **是否必填**：勾选后用户必须输入

#### 4.1.3 SQL安全限制

为防止SQL注入，系统禁止以下操作：
- INSERT、UPDATE、DELETE、DROP等写操作
- 存储过程调用
- 多语句执行（分号分隔）
- 危险函数（xp_cmdshell等）

### 4.2 报表查询

#### 4.2.1 查询报表

1. 导航至"报表管理" → "报表查询"
2. 选择要查询的报表
3. 输入查询参数（如有）
4. 点击"查询"按钮
5. 查看结果数据

#### 4.2.2 数据导出

支持两种导出格式：
- **Excel (.xlsx)**：适合数据分析和二次处理
- **CSV**：适合数据迁移和系统对接

导出步骤：
1. 查询出数据后点击"导出"按钮
2. 选择导出格式
3. 系统自动下载文件

### 4.3 数据源管理

#### 4.3.1 添加数据源

1. 导航至"系统管理" → "数据源管理"
2. 点击"新建数据源"
3. 填写连接信息：
   - **数据源名称**：便于识别
   - **数据库类型**：SQL Server/MySQL/PostgreSQL/Oracle/SQLite
   - **服务器地址**：数据库服务器IP或主机名
   - **端口**：数据库端口
   - **数据库名**：要连接的数据库
   - **用户名/密码**：数据库认证信息

4. 点击"测试连接"验证
5. 保存数据源

#### 4.3.2 数据源安全

- 密码使用AES加密存储
- 连接信息仅在服务器端保存
- 前端不暴露完整连接字符串

### 4.4 用户管理

#### 4.4.1 用户账户

| 字段 | 说明 |
|------|------|
| 用户名 | 登录账户，唯一不可重复 |
| 密码 | 至少8位，包含大小写字母和数字 |
| 真实姓名 | 用户显示名称 |
| 状态 | 启用/禁用 |
| 角色 | 分配的角色（决定权限） |

#### 4.4.2 创建用户

1. 导航至"系统管理" → "用户管理"
2. 点击"新建用户"
3. 填写用户信息
4. 分配角色
5. 保存

> **注意：** root用户是系统管理员，不可删除或禁用

### 4.5 角色权限

#### 4.5.1 角色管理

系统预设角色：
- **系统管理员**：拥有所有权限
- **报表管理员**：管理报表和数据源
- **报表查看者**：仅可查询报表

#### 4.5.2 权限列表

| 权限 | 说明 |
|------|------|
| 用户管理 | 创建、编辑、删除用户 |
| 角色管理 | 管理角色和权限分配 |
| 数据源管理 | 管理数据库连接 |
| 报表设计 | 创建、编辑、删除报表 |
| 报表查询 | 查询报表和导出数据 |
| 系统配置 | 系统设置和备份 |
| 日志查看 | 查看操作日志 |

### 4.6 许可证管理

#### 4.6.1 查看许可证状态

1. 导航至"系统管理" → "许可证管理"
2. 查看当前许可证信息：
   - 产品名称和版本
   - 机器码（用于申请许可证）
   - 许可证状态（有效/过期/未激活）
   - 到期时间
   - 已用/最大用户数

#### 4.6.2 试用期

- 新安装提供15天免费试用
- 试用期结束后需购买正式许可证

#### 4.6.3 激活许可证

1. 获取机器码
2. 将机器码发送至 `14279807@qq.com` 申请许可证
3. 收到许可证文件后，在系统中导入
4. 导航至"许可证管理" → "导入许可证"
5. 选择许可证文件并确认

### 4.7 系统备份

#### 4.7.1 创建备份

1. 导航至"系统管理" → "备份管理"
2. 点击"创建备份"
3. 选择备份内容：
   - 数据库数据
   - 配置文件
   - 报表定义
4. 点击"开始备份"
5. 下载备份文件

#### 4.7.2 恢复数据

1. 导航至"系统管理" → "备份管理"
2. 点击"恢复数据"
3. 选择备份文件
4. 确认恢复（此操作会覆盖现有数据）

> **警告：** 恢复操作不可逆，请谨慎操作！

### 4.8 操作日志

#### 4.8.1 查看日志

1. 导航至"系统管理" → "日志管理"
2. 可按条件筛选：
   - 时间范围
   - 操作用户
   - 操作类型
3. 查看操作详情

#### 4.8.2 日志内容

记录以下操作：
- 用户登录/登出
- 用户创建/修改/删除
- 报表创建/修改/删除
- 数据源配置变更
- 系统配置变更
- 数据备份/恢复

## 5. 常见问题

### 5.1 安装问题

**Q: 安装时提示"端口被占用"**
A: 检查8089端口是否被其他程序占用，或修改安装配置使用其他端口。

**Q: 服务启动失败**
A:
1. 检查.NET 8.0 Runtime是否正确安装
2. 查看Windows事件查看器中的错误日志
3. 确认数据库连接配置正确

### 5.2 登录问题

**Q: 忘记root密码**
A:
1. 停止DFAppService服务
2. 联系技术支持重置密码
3. 或重新初始化数据库

**Q: 登录后立即退出**
A:
1. 检查系统时间是否正确
2. 清除浏览器缓存
3. 检查许可证是否过期

### 5.3 报表问题

**Q: SQL查询超时**
A:
1. 优化SQL语句
2. 添加适当的索引
3. 减少查询数据量

**Q: 导出数据为空**
A:
1. 确认查询有返回数据
2. 检查浏览器是否阻止了下载

### 5.4 数据源问题

**Q: 连接测试失败**
A:
1. 确认数据库服务正在运行
2. 检查防火墙是否允许连接
3. 验证用户名密码正确
4. 确认网络连通性

### 5.5 许可证问题

**Q: 试用期提前结束**
A:
1. 检查系统时间是否被修改
2. 联系技术支持核实

**Q: 许可证激活失败**
A:
1. 确认许可证文件未损坏
2. 确认机器码与申请时一致
3. 联系技术支持
