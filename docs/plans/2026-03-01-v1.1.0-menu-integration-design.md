# V1.1.0 菜单集成设计文档

> **版本**: V1.1.0
> **创建日期**: 2026-03-01
> **状态**: 已批准并实施

## 1. 概述

### 1.1 目标
将 V1.1.0 的 Dashboard/Kanban 功能入口添加到系统侧边栏菜单，并配置相应的权限控制。

### 1.2 背景
V1.1.0 已实现以下功能模块：
- 看板管理 (`/dashboard`)
- 车间大屏配置 (`/display`)
- 全屏展示 (`/display/fullscreen`)

但侧边栏菜单缺少入口，用户无法直接访问这些功能。

## 2. 设计决策

### 2.1 菜单结构
**选择方案**：独立菜单项（顶级菜单）

```
侧边栏菜单结构：
├── 首页
├── 报表管理
│   ├── 报表查询
│   └── 报表设计
├── 看板管理 ← 新增
├── 车间大屏 ← 新增
├── 许可管理
└── 系统管理
    ├── 用户管理
    ├── 权限组管理
    ├── 数据源管理
    ├── 日志管理
    └── 备份管理
```

### 2.2 权限控制
使用现有权限系统：
- `dashboard:view` - 访问问看板管理
- `display:view` - 访问车间大屏配置

后端 DbInitializer.cs 已预定义这些权限。

## 3. 实现细节

### 3.1 前端修改

**文件**: `frontend/src/App.vue`

**改动 1**: 导入图标
```javascript
import { ..., DataBoard, Monitor } from '@element-plus/icons-vue'
```

**改动 2**: 添加菜单项
```vue
<!-- 看板管理 -->
<el-menu-item index="/dashboard" v-if="userStore.hasPermission('dashboard:view')">
  <el-icon><DataBoard /></el-icon>
  <span>看板管理</span>
</el-menu-item>

<!-- 车间大屏 -->
<el-menu-item index="/display" v-if="userStore.hasPermission('display:view')">
  <el-icon><Monitor /></el-icon>
  <span>车间大屏</span>
</el-menu-item>
```

### 3.2 CHANGELOG 更新

在 `CHANGELOG.md` 顶部添加 V1.1.0 版本记录。

### 3.3 数据库权限同步

后端重启时，`DbInitializer.SyncMissingPermissionsAsync()` 会自动将新权限同步到数据库。
