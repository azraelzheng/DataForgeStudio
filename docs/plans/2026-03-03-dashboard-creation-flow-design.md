# 大屏创建流程优化设计

> **目标**: 优化大屏创建流程，支持向导式创建、预置模板、发布状态管理和公开访问功能

## 1. 概述

### 1.1 背景

当前大屏创建流程较为简单，直接进入设计器后配置所有信息。用户期望一个更专业的工作流：
- 明确的草稿/已发布状态分离
- 创建时可选择预置模板
- 发布后支持公开访问（无需登录）
- 支持授权用户功能

### 1.2 成功标准

1. 用户可以通过向导对话框快速创建大屏
2. 创建时可选择预置模板，自动应用布局
3. 大屏有明确的状态（草稿/已发布）
4. 已发布的大屏可设置公开访问
5. 公开访问的大屏可通过URL在车间显示屏展示

## 2. 状态模型设计

### 2.1 大屏状态流转

```
┌─────────┐     发布      ┌─────────┐
│  草稿   │ ────────────→ │ 已发布  │
│ (draft) │ ←──────────── │(published)│
└─────────┘    取消发布    └─────────┘
                                │
                                │ 设为公开
                                ↓
                          ┌─────────┐
                          │  公开   │
                          │(public) │
                          └─────────┘
```

### 2.2 数据库字段变更

**Dashboard 表新增/修改字段：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Status | nvarchar(20) | 状态：draft / published，默认 draft |
| IsPublic | bit | 是否公开访问，默认 false |
| PublicUrl | nvarchar(50) | 公开访问URL标识（GUID短码） |
| AuthorizedUserIds | nvarchar(max) | 授权用户ID列表（JSON数组） |

### 2.3 权限控制逻辑

| 状态 | IsPublic | 访问权限 |
|------|----------|----------|
| draft | - | 仅创建者和管理员 |
| published | false | 登录用户 + AuthorizedUserIds 中的用户 |
| published | true | 任何人（无需登录） |

## 3. 前端设计

### 3.1 新建大屏向导

**触发方式**: 列表页点击"新建大屏"按钮

**对话框字段：**

| 分组 | 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| 基本信息 | name | 文本 | 是 | 大屏名称，最长50字符 |
| | description | 文本 | 否 | 描述，最长200字符 |
| 画布设置 | template | 选择 | 否 | 预置模板选择 |
| | size | 选择 | 是 | 画布尺寸：1920x1080/2560x1440/3840x2160/自定义 |
| | theme | 单选 | 是 | 主题：dark / light |
| | backgroundColor | 颜色 | 是 | 背景颜色，默认 #0a1628 |
| 数据刷新 | refreshInterval | 选择 | 是 | 自动刷新间隔：0/10/30/60/300/600秒 |

**预置模板列表：**

1. 生产监控 - 3列布局，表格+数字卡片
2. 质检 - 上下布局，表格+图表
3. 工序进度 - 表格+甘特图样式
4. 订单进度 - 多表格+趋势图
5. 设备状态 - 状态灯+设备列表
6. 看板布局 - 3列看板样式

### 3.2 列表页改造

**表格列调整：**

| 列名 | 宽度 | 说明 |
|------|------|------|
| 大屏名称 | 200px | - |
| 描述 | 150px | 超出省略 |
| 状态 | 80px | 标签：草稿(灰)/已发布(蓝)/公开(绿) |
| 组件数 | 70px | 居中对齐 |
| 创建时间 | 160px | - |
| 操作 | 280px | 根据状态显示不同按钮 |

**操作按钮逻辑：**

```
草稿状态:  [设计] [预览] [复制] [删除]
已发布:    [设计] [预览] [访问设置] [取消发布] [复制] [删除]
公开状态:  [设计] [预览] [访问设置] [取消发布] [复制链接] [复制] [删除]
```

### 3.3 访问设置对话框

发布后点击"访问设置"打开：

| 字段 | 类型 | 说明 |
|------|------|------|
| isPublic | 开关 | 是否开启公开访问 |
| publicUrl | 只读文本 | 公开链接，可复制 |
| authorizedUsers | 多选 | 授权用户列表（isPublic=false时显示） |

## 4. 后端 API 设计

### 4.1 新增/修改端点

```
POST   /api/dashboards
请求体增加: templateId (可选)

POST   /api/dashboards/{id}/publish
响应: { success, data: { status: "published" } }

DELETE /api/dashboards/{id}/publish
响应: { success, data: { status: "draft" } }

PUT    /api/dashboards/{id}/access
请求体: { isPublic: bool, authorizedUserIds: [] }
响应: { success, data: { publicUrl } }

GET    /api/public/d/{publicUrl}
无需认证，返回大屏展示数据
```

### 4.2 创建大屏请求体

```json
{
  "name": "生产监控大屏",
  "description": "车间生产数据监控",
  "width": 1920,
  "height": 1080,
  "theme": "dark",
  "backgroundColor": "#0a1628",
  "refreshInterval": 30,
  "templateId": "production-monitor"
}
```

### 4.3 模板布局定义

后端需定义预置模板的组件布局，创建时根据 templateId 自动添加组件：

```json
{
  "id": "production-monitor",
  "name": "生产监控",
  "layout": [
    { "x": 0, "y": 0, "w": 8, "h": 8, "type": "table", "title": "生产数据监控" },
    { "x": 8, "y": 0, "w": 4, "h": 4, "type": "card-number", "title": "当日产量" },
    { "x": 8, "y": 4, "w": 4, "h": 4, "type": "card-number", "title": "设备稼动率" }
  ]
}
```
