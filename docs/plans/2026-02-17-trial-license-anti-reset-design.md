# 试用许可证防重置机制设计

**日期**: 2026-02-17
**状态**: 已批准
**目标**: 防止用户通过卸载重装无限重置试用期

---

## 1. 背景

### 1.1 当前问题
- 试用许可证存储在数据库中（30天有效期）
- 卸载重装会清空数据库，导致可以无限重置试用期
- 现有检查仅依赖数据库记录

### 1.2 需求
- **部署环境**: 企业内部 Windows Server
- **离线要求**: 必须完全离线工作
- **防护级别**: 最高（商业软件级别）
- **试用期**: 15 天

---

## 2. 架构设计

### 2.1 架构概览

```
┌────────────────────────────────────────────────────────────┐
│                    LicenseService                           │
│  ┌──────────────────────────────────────────────────────┐  │
│  │              TrialLicenseTracker (新增)               │  │
│  │  ┌─────────────┬─────────────┬─────────────────────┐ │  │
│  │  │ 注册表存储   │ ProgramData │ 应用目录存储        │ │  │
│  │  │ HKLM\...    │ .dat文件    │ config\.cache      │ │  │
│  │  └─────────────┴─────────────┴─────────────────────┘ │  │
│  │                       ↓                               │  │
│  │              DPAPI 加密 (绑定机器)                     │  │
│  │                       ↓                               │  │
│  │              交叉验证 & 防篡改检测                     │  │
│  └──────────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────────┘
```

### 2.2 核心组件
- **TrialLicenseTracker**: 新增类，负责多位置存储和交叉验证
- **LicenseService**: 调用 TrialLicenseTracker 获取首次运行时间
- **DPAPI**: Windows 原生加密 API，绑定机器

## 3. 存储位置设计

| 位置 | 路径 | 伪装名称 | 说明 |
|------|------|----------|------|
| **注册表** | `HKLM\SOFTWARE\Microsoft\CryptoAPI\v2\machine` | `CacheData` | 伪装成系统加密缓存 |
| **ProgramData** | `C:\ProgramData\Microsoft\Crypto\RSA\MachineKeys\{GUID}.dat` | 随机GUID文件名 | 伪装成 RSA 密钥缓存 |
| **应用目录** | `{AppDir}\config\.runtime_cache` | 隐藏文件 | 应用运行时缓存 |

### 3.1 文件命名规则
- 注册表值名：`MachineGuid`（与现有系统值混淆）
- ProgramData 文件名：基于机器码生成的固定 GUID
- 应用目录文件：隐藏 + 系统属性

---

## 4. 数据结构

### 4.1 存储格式（JSON，DPAPI 加密）

```json
{
  "v": 1,
  "mc": "abc123def456...",
  "fr": "2024-01-15T10:30:00Z",
  "te": "2024-01-30T10:30:00Z",
  "cs": "sha256hash...",
  "nk": "randomnonce..."
}
```

### 4.2 字段说明

| 字段 | 含义 | 说明 |
|------|------|------|
| `v` | 版本号 | 数据结构版本，便于未来升级 |
| `mc` | 机器码 | 绑定当前机器 |
| `fr` | 首次运行时间 | UTC 时间，试用期起始点 |
| `te` | 试用到期时间 | UTC 时间，fr + 15天 |
| `cs` | 校验和 | `sha256(v + mc + fr + te + nk)` 防篡改 |
| `nk` | 随机数 | 增加校验和复杂度 |

### 4.3 加密范围
- 整个 JSON 字符串使用 DPAPI 加密
- DPAPI 绑定机器，无法在其他机器解密

## 5. 验证流程

### 5.1 启动验证流程

```
1. 读取三个位置的数据
   - 注册表 → 解密&校验
   - ProgramData → 解密&校验
   - 应用目录 → 解密&校验
   ↓
2. 收集有效的时间戳列表
   ↓
3. 判断情况：
   - [全有效&一致] → 正常使用
   - [部分有效] → 取最早时间，重写无效位置
   - [全无效] → 首次运行：生成新记录，写入3个位置
   ↓
4. 验证试用期
   - 当前时间 < 试用到期时间 ? 有效 : 过期
   ↓
5. 返回结果（有效/过期）
```

### 5.2 防篡改检测
- 校验和（cs）验证失败 → 视为无效
- 机器码（mc）不匹配 → 视为无效
- DPAPI 解密失败 → 视为无效

### 5.3 取最早时间逻辑
防止用户删除部分文件来"重置"，只要有任何一个位置保留了真实时间，就会使用它。

## 8. 安全特性总结

1. **DPAPI 绑定机器**: 加密数据只能在同一台机器解密
2. **多位置冗余**: 3个位置存储，删除部分仍可检测
3. **取最早时间**: 防止删除重置
4. **自动修复**: 检测不一致时自动重写
5. **伪装命名**: 类似系统组件，增加定位难度
6. **校验和防篡改**: 任何修改都会导致校验失败
