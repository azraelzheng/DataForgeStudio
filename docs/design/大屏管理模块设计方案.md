### 2.2 数据模型 (Data Model)
#### 2.2.1 数据库表结构(Database Schema)
##### Dashboards 大屏主表
```sql
-- 大屏主表
CREATE TABLE [Dashboards] (
    [DashboardId] INT IDENTITY(1,1),    -- 大屏ID，主键
    [Name] NVARCHAR(100) NOT NULL,        -- 大屏名称
    [Description] NVARCHAR(500) NULL,     -- 大屏描述
    [Theme] VARCHAR(20) DEFAULT 'dark',            -- 主题： dark/light
    [Width] INT DEFAULT 1920,                  -- 画布宽度
    [Height] INT DEFAULT 1080,                 -- 画布高度
    [BackgroundColor] VARCHAR(20) DEFAULT '#0a1628', -- 背景颜色
    [BackgroundImage] VARCHAR(500) NULL,    -- 背景图片URL
    [RefreshInterval] INT DEFAULT 30,        -- 自动刷新间隔(秒)
    [LayoutConfig] NVARCHAR(MAX) NULL,           -- 布局配置 (JSON)
    [ThemeConfig] NVARCHAR(max) NULL,           -- 主题配置 (Json)
    [Status] VARCHAR(20) DEFAULT 'draft',      -- 状态: draft/published
    [IsPublic] BIT DEFAULT 0,                 -- 是否公开
    [PublicUrl] VARCHAR(50) NULL,             -- 公开访问URL标识
    [AuthorizedUserIds] NVARCHAR(MAX) NULL, -- 授权用户ID列表(JSON)
    [CreatedBy] INT NULL,                -- 创建人ID
    [CreatedTime] DATETIME DEFAULT GETdate(), -- 创建时间
    [UpdatedTime] DATETIME NULL,             -- 更新时间
    PRIMARY KEY ([DashboardId])
);

-- 大屏组件表
CREATE TABLE [DashboardWidgets] (
    [WidgetId] INT IDENTITY(1,1),    -- 组件ID
    [DashboardId] INT NOT NULL,                -- 所属大屏ID
    [ReportId] INT NOT NULL,                 -- 关联报表ID
    [WidgetType] VARCHAR(50) NOT NULL,      -- 组件类型
    [Title] VARCHAR(100) NULL,              -- 组件标题
    [PositionX] INT DEFAULT 0,              -- X坐标
    [PositionY] INT DEFAULT 0,              -- Y坐标
    [Width] INT DEFAULT 4,                -- 宽度
    [Height] INT DEFAULT 4,                -- 高度
    [DataConfig] NVARCHAR(MAX) NULL,        -- 数据配置 (JSON)
    [StyleConfig] NVARCHAR(MAX) NULL,        -- 样式配置 (JSON)
    [CreatedTime] DATETIME DEFAULT getdate(), -- 创建时间
    PRIMARY KEY ([WidgetId]),
    FOREIGN KEY ([DashboardId]) REFERENCES [Dashboards]([DashboardId]),
    FOREIGN KEY ([ReportId]) REFERENCES [Reports]([ReportId])
);

-- 组件规则表
CREATE TABLE [WidgetRules] (
    [RuleId] INT IDENTITY(1,1),       -- 规则ID
    [WidgetId] INT NOT NULL,          -- 组件ID
    [RuleName] VARCHAR(100) NOT NULL, -- 规则名称
    [Field] VARCHAR(100) NOT NULL,    -- 字段名
    [Operator] VARCHAR(20) NOT NULL,  -- 操作符: eq/ne/gt/lt/gte/lte/between/contains
    [Value] NVARCHAR(200) NULL,         -- 比较值
    [ActionType] VARCHAR(50) NOT NULL, -- 动作类型: highlight/color/icon/animation
    [ActionValue] NVARCHAR(500) NULL,  -- 动作值
    [Priority] INT DEFAULT 0,           -- 优先级
    [CreatedTime] DATETIME DEFAULT getdate(), -- 创建时间
    PRIMARY KEY ([RuleId]),
    FOREIGN KEY ([WidgetId]) REFERENCES [DashboardWidgets]([WidgetId])
);
```

#### 2.2.2 实体类定义 (Entity Classes)
##### Dashboard 实体
```csharp
namespace DataForgeStudio.Domain.Entities;

[Table("Dashboards")]
public class Dashboard
{
    [Table("Dashboards")]
    [Key]
    [DatabaseGenerated(DatabaseGeneratedOption.Identity)]
    public int DashboardId { get; set; }

    [Required]
    [MaxLength(100)]
    public string Name { get; set; } = string.Empty;

    [MaxLength(500)]
    public string? Description { get; set; }

    [MaxLength(20)]
    public string Theme { get; set; } = "dark";

    public int RefreshInterval { get; set; } = 30;

    public bool IsPublic { get; set; } = false;

    public string? LayoutConfig { get; set; }

    public string? ThemeConfig { get; set; }

    [MaxLength(20)]
    public string Status { get; set; } = "draft";

    [MaxLength(50)]
    public string? PublicUrl { get; set; }

    public string? AuthorizedUserIds { get; set; }

    public int Width { get; set; } = 1920;

    public int Height { get; set; } = 1080;

    [MaxLength(20)]
    public string BackgroundColor { get; set; } = "#0a1628";

    [MaxLength(500)]
    public string? BackgroundImage { get; set; }

    public int? CreatedBy { get; set; }

    public DateTime CreatedTime { get; set; } = DateTime.UtcNow;

    public DateTime? UpdatedTime { get; set; }

    public virtual ICollection<DashboardWidget> Widgets { get; set; } = new List<DashboardWidget>();

    [ForeignKey(nameof(CreatedBy))]
    public virtual User? Creator { get; set; }
}
```

### 2.3 API 设计 (API Design)
#### 大屏管理 API
```
GET /api/dashboards               # 获取大屏列表
GET /api/dashboards/{id}              # 获取大屏详情
POST /api/dashboards                  # 创建大屏
PUT /api/dashboards/{id}              # 更新大屏
DELETE /api/dashboards/{id}           # 删除大屏
POST /api/dashboards/{id}/publish    # 发布大屏
DELETE /api/dashboards/{id}/publish   # 取消发布
PUT /api/dashboards/{id}/access     # 更新访问设置
GET /api/dashboards/{id}/access     # 获取访问设置
```
#### 组件管理 API
```
POST /api/dashboards/{id}/widgets              # 添加组件
Put /api/dashboards/{id}/widgets/{widgetId}  # 更新组件
DELETE /api/dashboards/{id}/widgets/{widgetId}  # 删除组件
PUT /api/dashboards/{id}/widgets/positions  # 批量更新组件位置
```
#### 数据获取 API
```
GET /api/dashboards/{id}/data             # 获取大屏数据（所有组件数据)
```
#### 一键转换 API
```
POST /api/dashboards/convert             # 从报表转换为大屏
```
#### 公开访问 API
```
GET /api/public/dashboards/{publicUrl}  # 公开大屏详情
GET /api/public/dashboards/{publicUrl}/data  # 公开大屏数据
```

## 四、数据源集成方案 (Data Source Integration)
### 4.1 报表调用方案（核心特性）
#### 概述
大屏组件通过关联已有报表来获取数据，而非直接编写 SQL。
**核心优势**：
- 复用现有报表逻辑
- 统一的数据格式和权限控制
- 支持参数化查询
- 便于维护和扩展

#### 数据流程
```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  大屏组件   │ ──▶ │  报表服务   │ ──▶ │   数据源    │
│  (Widget)   │     │ (ReportService)│   │ (DataSource)│
└─────────────┘     └─────────────┘     └─────────────┘
       │                   │
       ▼                   ▼
 ┌─────────────────────────────────────────────────────┐
│ DataConfig: {                                        │
│   reportId: 1,                                       │
│   queryConditionValues: {                            │
│     startDate: '2024-01-01',                         │
│     endDate: '2024-12-31'                            │
│   }                                                  │
│ }                                                    │
└─────────────────────────────────────────────────────┘
```

### 4.2 组件数据配置结构
```typescript

  reportId: number;                    // 关联的报表ID
  queryConditionValues?: {             // 查询条件值
    [key: string]: any;
  };
  dataMapping?: {                    // 数据字段映射
    xField: string;                   // X轴字段

  };

```

### 4.3 数据获取流程
1. **获取大屏数据**: 调用 `GET /api/dashboards/{id}/data`
2. **并行获取**: 后端并行获取所有组件的数据
3. **数据返回**: 返回每个组件的数据结果
4. **前端渲染**: 组件根据返回的数据进行渲染

### 4.4 数据刷新机制
- **自动刷新**: 根据大屏配置的 RefreshInterval 定时刷新
- **手动刷新**: 用户点击刷新按钮
- **实时更新**: WebSocket 推送（可选）

### B. 数据获取示例
```typescript
// useDataSource.ts
import { ref, onUnmounted } from 'vue'
import request from '@/api/request'

export function useWidgetData(widget: DashboardWidget) {
  const data = ref<any[]>([])
  const loading = ref(false)
  const error = ref<string | null>(null)

  const fetchData = async () => {
    const config = parseDataConfig(widget.dataConfig)
    if (!config.reportId) {
      data.value = []
      return
    }

    loading.value = true
    try {
      const res = await request.post(`/api/reports/${config.reportId}/execute`, {
        parameters: config.queryConditionValues || {}
      })
      if (res.success) {
        data.value = res.data || []
      } else {
        error.value = res.message
      }
    } catch (e) {
      error.value = '数据加载失败'
    } finally {
      loading.value = false
    }
  }

  fetchData()

  return { data, loading, error, refresh: fetchData }
}
```
