================================================================================
                    DataForgeStudio 安装说明
================================================================================

【系统要求】

操作系统：
  ✅ Windows 10 或更高版本
  ✅ Windows Server 2012 R2 或更高版本
  ✅ Windows Server 2016 或更高版本（推荐）

  ❌ 不支持的系统：
     - Windows 7 / 8 / 8.1
     - Windows Server 2008 / 2008 R2
     - Windows Server 2012（已停止支持）

运行环境：
  - .NET 8.0 Runtime（安装程序会自动检测并提示安装）
  - 至少 4GB 内存
  - 至少 500MB 可用磁盘空间

重要提示：
  .NET 8.0 不支持 Windows Server 2008 R2 及更早版本。
  如需在这些系统上运行，请升级操作系统。

数据库：
  - SQL Server 2005 或更高版本
  - MySQL 5.7 或更高版本
  - PostgreSQL 10 或更高版本
  - Oracle 11g 或更高版本
  （以上任选其一，推荐 SQL Server）


【安装步骤】

1. 运行安装程序
   双击 DataForgeStudio-Setup.exe 启动安装向导

2. 选择安装目录
   默认目录：C:\Program Files\DataForgeStudio
   建议使用默认目录

3. 配置数据库
   安装完成后，运行系统管理工具配置数据库连接

4. 启动服务
   - 通过系统管理工具启动服务
   - 或使用 Windows 服务管理器启动


【首次登录】

安装完成后，系统会生成 root 用户的临时密码：
  - 用户名：root
  - 密码：查看安装日志或服务启动时的控制台输出

重要：首次登录后请立即修改密码！


【端口配置】

默认端口：
  - API 服务：5000
  - Web 服务：80

如需修改，请编辑配置文件：
  - API 配置：Server/appsettings.json
  - Web 配置：WebServer/conf/nginx.conf


【防火墙设置】

确保以下端口在防火墙中开放：
  - 5000（API 服务）
  - 80（Web 服务，或您配置的其他端口）


【常见问题】

Q: 安装后无法访问？
A: 1. 检查服务是否启动
   2. 检查防火墙设置
   3. 检查端口是否被占用

Q: 数据库连接失败？
A: 1. 确认 SQL Server 服务已启动
   2. 检查连接字符串
   3. 确认 SQL Server 允许远程连接

Q: 忘记密码怎么办？
A: 联系系统管理员重置密码

Q: 点击"测试端口"后系统蓝屏/重启？
A: 这通常是由于第三方安全软件驱动冲突导致的。
   已知冲突的软件：
   - 腾讯电脑管家
   - 腾讯QQ安全组件
   - 360安全卫士
   建议：暂时禁用或卸载这些软件后再进行配置。


【已知兼容性问题】

某些安全软件的系统监控驱动可能与本软件产生冲突：
- 症状：端口测试或服务启动时系统蓝屏
- 原因：安全软件的内核驱动与 .NET 网络操作冲突
- 解决：卸载或禁用冲突的安全软件，改用 Windows Defender


【卸载】

1. 停止所有服务
2. 通过 Windows 控制面板卸载
3. 手动删除数据目录（可选）：
   - C:\ProgramData\DataForgeStudio
